Replit Agent Prompt — Update SignTakeoff App
Context
I have an existing Node.js/TypeScript SignTakeoff app that currently sends full plan PDFs to the Gemini API for sign takeoff analysis. I want to restructure it to use a cost-efficient pipeline where most work is done in deterministic code and only 2-3 targeted Gemini API calls are made per takeoff. This will reduce cost from ~$4-6 per takeoff to ~$0.01-0.03.
Current state
Node.js/TypeScript app
Gemini API backend (keep this — do not switch to a different LLM)
Full workflow: upload PDF → takeoff → marked-up PDF → export
Currently sends many/all plan pages to Gemini (expensive)
What to change — the pipeline architecture
Restructure the app into a 7-step pipeline. Steps 3 and 4b call Gemini. All other steps run as deterministic Node.js code with NO LLM calls.
Step 1 — PDF Intake (deterministic)
Use pdf-lib or pdf-parse (npm) to extract:

Page count
Metadata (title, author, creation date)
Text from page 1 only (the drawing index)

Step 4a — Room Label Extraction (deterministic)
This is the most important deterministic step. Use pdf.js (pdfjs-dist npm) or pdf-parse to extract all text with positions from each floor plan page.

interface RoomLabel {

  roomNumber: string;       // e.g., "109"

  roomName: string;         // e.g., "POLICE CHIEF"

  level: string;            // e.g., "LEVEL 1"

  sheetId: string;          // e.g., "A-101"

  pdfPage: number;          // e.g., 25

  coords: { x: number; y: number };  // normalized 0-1000

  bbox: { x0: number; y0: number; x1: number; y1: number };

}

Algorithm:

Get all text items with positions from the PDF page
Find items matching room number pattern: /^[1-9]\d{2}[A-Z]?$/
For each number, look for CAPS text directly above (within ~40pt vertical, ~100pt horizontal)
Use fuzzy matching for abbreviations:

const SYNONYMS: Record<string, string[]> = {

  'VEST': ['VEST', 'VESTIBULE'],

  'EMERG': ['EMERG', 'EMERGENCY'],

  'INTERROG': ['INTERROG', 'INTERROGATION'],

  'CORR': ['CORR', 'CORRIDOR'],

};

Filter out numbers in the legend/notes area (typically right 20% of page)
Normalize coordinates to 0-1000 grid:

const normX = Math.round(((pdfX - planLeft) / planWidth) * 1000);

const normY = Math.round((1 - (pdfY - planBottom) / planHeight) * 1000);  // flip Y for bottom-left origin

Step 4c — Build Room Inventory (deterministic)
Merge the room labels from 4a with the occupant loads from 4b. Derive flags for each room:

interface RoomInventory extends RoomLabel {

  occupantLoad: number | null;

  occupancyGroup: string | null;

  flags: {

    isRestroom: boolean;    // TOILET, BATH, SHOWER, WC, RESTROOM, LAVATORY

    isStair: boolean;       // starts with STAIR

    isElevator: boolean;    // ELEV, ELEVATOR

    isVestibule: boolean;   // VEST, VESTIBULE

    isCorridorOrHall: boolean;  // HALL, CORR, CORRIDOR, LOBBY, FOYER

    isVehicleBay: boolean;  // APPARATUS, VEHICLE BAY, SALLY PORT, GARAGE

    isMepUnoccupied: boolean;  // MECHANICAL, ELECTRICAL, etc. AND occupantLoad == 0

    isVariableUse: boolean; // TRAINING, COMMUNITY, EOC, MULTIPURPOSE

    isPublicFacing: boolean; // LOBBY, RECEPTION, WAITING

    isStaffOnly: boolean;   // "STAFF" in name

    isAssembly: boolean;    // occupancyGroup in ['A-2', 'A-3'] OR occupantLoad >= 50

  };

}
Step 5 — Apply Rules R1-R15 (deterministic)
This is pure TypeScript logic. No LLM call needed.

function applyRules(room: RoomInventory, plaqueSchedule: PlaqueSchedule): TakeoffEntry[] {

  const signs: TakeoffEntry[] = [];

  

  // R1: Default Room ID

  if (!room.flags.isCorridorOrHall && !room.flags.isVehicleBay && 

      !room.flags.isMepUnoccupied && !room.flags.isStair && !room.flags.isElevator) {

    

    // R2: Variable use → insert variant

    if (room.flags.isVariableUse) {

      const qty = room.flags.isAssembly ? 2 : 1;  // dual function = 2

      signs.push({ signType: 'Room ID w/insert', qty, ruleRef: 'R2', 

                   color: '#7c3aed', plaqueTypeId: findPlaque('insert', plaqueSchedule) });

    } else {

      signs.push({ signType: 'Room ID', qty: 1, ruleRef: 'R1',

                   color: '#2563eb', plaqueTypeId: 'A' });

    }

  }

  

  // R8: Restroom

  if (room.flags.isRestroom) {

    const variant = selectRestroomVariant(room);

    signs.push({ signType: 'Restroom', qty: 1, ruleRef: 'R8',

                 color: '#14b8a6', plaqueTypeId: variant });

  }

  

  // R9: Exit

  if (room.flags.isVestibule || 

      (room.flags.isPublicFacing && room.roomName.includes('LOBBY'))) {

    signs.push({ signType: 'Exit', qty: 1, ruleRef: 'R9', color: '#dc2626' });

  }

  if (room.flags.isAssembly && (room.occupantLoad ?? 0) >= 50) {

    signs.push({ signType: 'Exit', qty: 2, ruleRef: 'R9', color: '#dc2626' });

  }

  

  // R10: Capacity

  if (room.flags.isAssembly) {

    signs.push({ signType: 'Max Occupancy', qty: 1, ruleRef: 'R10', color: '#f59e0b' });

  }

  

  // R14: Office Directory (corridors/halls only)

  if (room.flags.isCorridorOrHall && room.flags.isPublicFacing) {

    signs.push({ signType: 'Office Directory', qty: 1, ruleRef: 'R14', color: '#3b82f6' });

  }

  

  return signs;

}

Key principle: the Gemini system prompt is SHORT
Do NOT put the full rulebook in the Gemini system prompt. The rules engine runs in TypeScript. Gemini is only used for:

Reading the plaque schedule from a rasterized page (structured extraction)
Reading the occupant loads table from a rasterized page (structured extraction)
Optionally resolving ambiguous room classifications

Each Gemini call gets a focused, task-specific prompt (~200 words) not the full methodology document. The methodology lives in your TypeScript code.

Once validated, remove the old Gemini-heavy flow
