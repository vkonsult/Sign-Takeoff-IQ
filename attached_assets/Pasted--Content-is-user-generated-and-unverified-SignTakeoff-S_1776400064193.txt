Good question — this is the weakest part of the spec as written. "Search the drawing index for sheet titles containing [keywords]" sounds deterministic, but on a real 100+ page plan set it falls apart for several reasons. Let me lay out what actually needs to happen, what rules are missing, and what I'd add.

Why the current Phase 2 is under-specified
The existing rule is essentially a keyword match against sheet titles in the drawing index. Problems:

The drawing index itself isn't always on page 1. Large sets often have a cover sheet, then a separate sheet index on G-001 or T-001. Some sets split the index across multiple sheets (architectural index, MEP index, civil index).
Sheet titles lie or abbreviate. "LIFE SAFETY PLAN" might be titled "CODE PLAN," "EGRESS PLAN," "EXITING PLAN," or just sit inside a sheet called "GENERAL NOTES." Occupant load tables frequently live on a sheet titled something like "A-003 CODE COMPLIANCE" with no "egress" keyword anywhere.
Signage schedules hide. The prompt correctly notes they often live in A-7XX millwork/interiors sheets, but they can also be on a single page inside A-000 general notes, inside the specs (Division 10 14 00), or on an interior elevation sheet with no "sign" in the title.
Keyword collisions. "EXIT" matches exit stair details, exit sign schedules, and exterior elevations depending on tokenizer. "PLAN" matches everything.
Sheet numbering conventions vary. A-1XX for floor plans is common but not universal — some offices use A-101/A-102 for first floor, A-201/A-202 for second floor (the hundreds digit is the level), others use A-101 through A-105 all on level 1. You can't assume numbering tells you level without checking.
Sheets get reissued. A 100-page set might have A-003, A-003.1, A-003R2. The index should reflect the latest, but doesn't always.
Rules I would add to Phase 2
Here's a revised Phase 2 that's actually robust on a 100-page set:

P2.1 — Locate the drawing index, don't assume page 1.

Open the table of contents / bookmarks in the PDF first. If the PDF has bookmarks, the sheet index is almost always there and gives you page-number-to-sheet-number mapping for free.
If no bookmarks: scan pages 1–5 for a page titled "DRAWING INDEX," "SHEET INDEX," "LIST OF DRAWINGS," or "TABLE OF CONTENTS." If not found there, scan the first page of each discipline section (cover sheets are often G-001, T-001, A-000, or the first A-sheet).
Record the full index as a structured list: sheet_number → sheet_title → pdf_page_number. This mapping is the foundation for everything downstream.
P2.2 — Classify every sheet, don't just search for keywords. Walk the entire index once and tag each sheet into one of these buckets using a priority-ordered ruleset (first match wins):

Priority	Bucket	Matches if sheet title contains (case-insensitive)	Or sheet number matches
1	signage_schedule	"signage", "sign schedule", "plaque", "signage details", "sign types"	—
2	life_safety	"life safety", "egress", "code plan", "code compliance", "exiting", "occupant load"	—
3	floor_plan	"floor plan", "overall plan", "enlarged plan" AND not "reflected", "ceiling", "roof", "foundation", "demo"	A-1XX, A-2XX where level prefix matches
4	general_notes	"general notes", "abbreviations", "symbols", "mounting heights", "typical details"	A-000, A-001, G-series
5	millwork_interiors	"millwork", "casework", "interior elevations", "interior details"	A-7XX, A-8XX
6	accessibility	"accessibility", "ADA", "MAAB", "CBC 11B", "barrier-free"	—
7	specifications	"specifications", "specs"	—
8	other	—	—
Negative keywords matter: "REFLECTED CEILING PLAN," "ROOF PLAN," "FOUNDATION PLAN," and "DEMOLITION PLAN" are NOT floor plans for this purpose and should be excluded even though they match "plan."

P2.3 — Confirm classification with a page-level check, not just the index title. Sheet titles in the index occasionally disagree with what's actually on the page (reissues, retitled sheets, scope changes mid-project). For each sheet in the three highest-value buckets (signage_schedule, life_safety, floor_plan), open the page and confirm:

Signage schedule: page contains a table with columns resembling "type / description / size / mounting" and/or sign-type identifiers (A, B, C… or S1, S2…)
Life safety: page contains an occupant load table OR egress-labeled plan with travel distance arrows/exit capacity calcs
Floor plan: page contains rooms with name/number labels at the scale of the building (not an enlarged detail)
If the page-level check fails, demote the sheet and flag it.

P2.4 — Find the signage schedule even when no sheet is titled "signage." Fallback search order if P2.2 didn't find a signage_schedule sheet:

Search sheet keynotes/legends on floor plans for "see sign schedule on sheet XX" or "S-1, S-2" tag references — trace to the referenced sheet
Scan A-000 and general-notes sheets for a sign schedule table
Scan A-7XX millwork sheets page by page for a schedule table
Check the specifications index for Division 10 14 00 (Signage) — the schedule may be in specs rather than on drawings
If all of the above fail, stop and ask the user for the signage spec/schedule rather than guessing
P2.5 — Map floor plans to levels explicitly. Don't infer level from sheet number. For each sheet classified as floor_plan, read the sheet title ("LEVEL 1 FLOOR PLAN," "FIRST FLOOR PLAN," "GROUND FLOOR PLAN," "MEZZANINE PLAN") and record level explicitly. If a sheet shows multiple levels (small buildings), record all of them. If the title is ambiguous, open the page and look for a level indicator in the title block.

P2.6 — Handle enlarged plans and plan-match-lines. On large buildings, floor plans get split: "A-101 LEVEL 1 - AREA A," "A-102 LEVEL 1 - AREA B," with a key plan showing how they fit. Treat each as a separate floor_plan sheet tagged with the same level and an area field. The key plan sheet itself is usually titled "KEY PLAN" or "OVERALL PLAN" and tells you how many area sheets to expect — use it as a checklist. If you've processed three area sheets but the key plan shows four areas, you're missing one.

P2.7 — Capture sheet revision status. Record the revision number/date from each sheet's title block for every sheet you're using. If you see a superseded sheet (lower revision than another copy in the set), use only the latest. Flag any case where two sheets have the same number with different revisions.

P2.8 — Output a reference-sheet manifest before proceeding to Phase 3. Before doing any extraction, emit:

Signage schedule: A-702 (PDF pg 84), rev 3, 2026-02-14
Life safety:     A-003 (PDF pg 7), rev 2, 2026-01-20
Floor plans:     A-101 Level 1 Area A (PDF pg 12)
                 A-102 Level 1 Area B (PDF pg 13)
                 A-201 Level 2 (PDF pg 20)
                 A-301 Mezzanine (PDF pg 28)
General notes:   A-000 (PDF pg 4)
Accessibility:   G-002 (PDF pg 2)
This manifest is the user's first checkpoint. If they spot a missing sheet, they catch it before any takeoff work starts — which on a 100+ page set saves hours.

What I'd also add as a separate principle
There should be an explicit rule that says: never assume a sheet exists by convention. If you need a signage schedule and none of the detection passes find one, ask — do not fabricate one or apply generic rules blindly. Phase 3 currently reads the signage sheet to build the plaque type list; if that sheet doesn't exist and no one notices, the entire downstream takeoff is running on the AI's assumptions about what plaque types "should" be, which is exactly the failure mode these kinds of tools are rightly criticized for.

Bottom line
The current Phase 2 is roughly three sentences. For a 100+ page set it needs to be a proper discovery phase with (1) an index-first pass, (2) bucketed classification with priority-ordered rules and negative keywords, (3) page-level verification of high-value sheets, (4) explicit fallback paths for missing signage schedules, (5) explicit level mapping, (6) handling for area-split plans, (7) revision handling, and (8) a manifest checkpoint before downstream phases run.
